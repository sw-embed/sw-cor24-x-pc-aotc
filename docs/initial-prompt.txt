== WHAT THIS PROJECT IS ==

sw-cor24-x-pc-aotc is an AOT compiler that translates p-code bytecode
(produced by the Pascal compiler p24p) into COR24 .s assembly files.
The output feeds into the existing assembler/linker toolchain to produce
native COR24 binaries — no p-code interpreter needed at runtime.

The x- prefix means this is a cross-tool written in Rust, running on
the host (not on COR24 hardware).

Key design decisions already locked in (from research.txt):
- Written in Rust, runs on host machine
- AOT first, JIT deferred to far-future
- Direct lowering first (p-code → .s), graduate to IR-based later
- Differential testing against the p-code interpreter (pv24a) is the
  core validation strategy
- Share semantic specs and test cases with the interpreter, NOT code
- Runtime helpers (write_int, heap_alloc, bounds_check, etc.) are
  called from generated code, not inlined
- Interpreter remains the canonical reference forever

Recommended Rust crate structure (from research.txt):
- pcode-model: opcode definitions, decoded instructions, procedure metadata
- pcode-analyze: CFG, stack-depth analysis, validation
- pcode-lower: lowering from p-code to simple IR
- cor24-emit-asm: COR24 assembly emission from lowered IR
- pcode-aotc: CLI binary

== KEY TECHNICAL DETAILS ==

COR24 register constraints (only 3 working registers):
- r0, r1, r2: scratch/expression (r0 = return value)
- r3 (fp): frame pointer
- r4 (sp): stack pointer
- r5 (z): zero register
- r6 (iv): interrupt vector
- r7 (ir): interrupt return

Register allocation strategy: TOS caching in r0, NOS optionally in
r1/r2, everything else in memory (frame slots / eval stack).

Runtime helpers are called (not inlined) for: integer output, character
output, string operations, heap alloc/free, bounds checking, set
operations, error reporting.
